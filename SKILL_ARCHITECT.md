This skill helps an architect analyze requirements, create a technical architecture, select patterns, and align engineering teams around a scalable implementation.

## When to use
- defining application architecture for a new project or major feature
- translating requirements into technical design and system boundaries
- aligning backend, frontend, DevOps, SecOps, and UX teams around architecture decisions

## Workflow
1. Assess project context and constraints
   - review requirements, business goals, and user outcomes
   - identify existing systems, platforms, and architectural constraints
   - capture security, compliance, performance, and operational needs
2. Define architecture goals and principles
   - establish quality attributes such as scalability, reliability, maintainability, and security
   - choose architectural patterns and integration approaches
   - document tradeoffs, assumptions, and non-functional priorities
3. Design the solution components
   - define system boundaries, services, modules, and data flows
   - select technology stacks, APIs, and integration points
   - identify deployment, infrastructure, and runtime requirements
4. Validate and align with teams
   - review the architecture with backend, frontend, DevOps, SecOps, and UX/UI experts
   - capture feedback, potential risks, and alternative approaches
   - finalize the architecture decision record and next steps
5. Prepare implementation readiness
   - create component-level tasks, design artifacts, and handoff notes
   - define acceptance criteria for architecture validation
   - ensure the architecture supports the planned sprint and delivery cadence

## Completion criteria
- a clear architecture vision and principle set
- documented system boundaries, components, and data flow
- selected technologies and integration strategy
- identified risks, assumptions, and tradeoffs
- alignment across engineering and operations teams

## Questions to ask
- What are the most important non-functional requirements?
- What existing systems or infrastructure must this architecture integrate with?
- What are the expected traffic, data, and performance constraints?
- What security, compliance, or operational requirements must be met?
- Which teams need to review and approve the architecture decisions?
